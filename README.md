# Alt-AltTab

[![CI](https://github.com/cm-kitagawa-zempei/alt-alttab/actions/workflows/ci.yml/badge.svg)](https://github.com/cm-kitagawa-zempei/alt-alttab/actions/workflows/ci.yml)

AltTab 代替の Cmd+Tab ウィンドウスイッチャー。ウィンドウ単位で切り替え、サムネイル表示に対応しています。

## 特徴

- アプリ単位ではなくウィンドウ単位での Cmd+Tab 切り替え
- ウィンドウのサムネイル表示（画面収録の許可時）
- 最小化中のウィンドウ・非表示アプリのウィンドウも切り替え対象にできる
- 切り替え中に W（閉じる）/ H（隠す）/ Q（終了）でウィンドウ・アプリを操作
- メニューバーから開く設定画面
- Dock に表示されない常駐アプリ

## 動作環境

- アプリの動作は macOS 14 以降（開発・動作確認は Apple Silicon + macOS 26。Intel Mac は未検証です）
- ビルドには Swift 6.2 以降の Xcode Command Line Tools が必要です。これがインストールできるのは **macOS 15.6 以降**なので、ソースからビルドして使う場合は実質こちらが下限になります。入っていない場合は次でインストールします。

  ```sh
  xcode-select --install
  ```

  Xcode 本体は不要です。Nix は開発用のオプションで、必須ではありません。

## インストール

```sh
git clone <このリポジトリのURL>
cd alt-alttab
make run
```

`build/Alt-AltTab.app` がビルドされ、そのまま起動します。

日常的に使う場合は `make install` を実行してください。`/Applications` にコピーしてそちらを起動します。ログイン時に自動起動したい場合は「システム設定 > 一般 > ログイン項目」に `/Applications/Alt-AltTab.app` を追加してください。

アップデートする場合は次の手順です。

```sh
git pull
make install
```

## 権限設定

- アクセシビリティ：必須です。キーボードイベントの監視に使用します。
- 画面収録：任意です。ウィンドウのサムネイル表示に使用します。許可しない場合はアイコンとタイトルのみの表示になります。

どちらも「システム設定 > プライバシーとセキュリティ」の該当ペインで許可してください。メニューバーの ⇥ アイコンから直接該当ペインを開くこともできます。

画面収録は許可した後にアプリの再起動が必要です。

メニューバーの ⇥ からは、アクセシビリティ・画面収録・イベントタップの状態確認と、設定画面（⌘,）を開けます。

### 署名用の証明書を作っておく

更新しながら使うなら、最初にやっておくことをおすすめします。

このアプリは配布バイナリを持たず、更新も `git pull` からのビルドで行います。証明書なしの ad-hoc 署名のままだとビルドのたびにバイナリの識別情報が変わるため、更新するたびに許可した権限が無効になり、`make reset-tcc` と再許可が必要になります。

Keychain Access で自己署名のコード署名証明書 `alt-alttab-dev` を一度作っておけば、Makefile がそれを自動検出して署名に使うようになり、以降は更新しても権限が持続します。作成手順は [DEVELOPMENT.md の「署名と TCC」](DEVELOPMENT.md#署名と-tcc) を参照してください（初回に一度だけの作業です）。

## 使い方

Cmd+Tab でスイッチャーを開き、Cmd を押したまま次のキーで操作します。

| キー | 動作 |
| --- | --- |
| Tab / → | 次のウィンドウへ |
| Shift+Tab / ← | 前のウィンドウへ |
| W | 選択中のウィンドウを閉じる |
| H | 選択中のウィンドウのアプリを非表示にする |
| Q | 選択中のウィンドウのアプリを終了する |
| Esc | キャンセル |
| Cmd を離す | 選択中のウィンドウに切り替え |

マウスでのホバーで選択、クリックでその場で切り替えることもできます。

W / H / Q はセッションを開いたまま続けて実行でき、対象がなくなるとセッションは自動的に終了します。

## 設定

メニューバーの ⇥ → 設定… から次の4項目を切り替えられます。

- 最小化中のウィンドウを表示：オフにすると最小化中のウィンドウをスイッチャーの対象から外します。
- 非表示アプリのウィンドウを表示：オフにすると非表示（Cmd+H）にしたアプリのウィンドウを対象から外します。
- 最小化中のウィンドウを暗く表示：最小化中のウィンドウのサムネイルを暗く表示します。
- ログイン時に自動起動：`/Applications` にインストールした状態（`make install`）での利用を想定しています。

変更は次に開く Cmd+Tab セッションから反映されます。

## トラブルシューティング

- メニューバーに ⇥ が見当たらない場合はアプリが起動していません。`/Applications/Alt-AltTab.app` を起動してください。設定の「ログイン時に自動起動」をオンにしておくと再発を防げます。
- 更新（再ビルド）後に Cmd+Tab が反応しなくなった場合は `make reset-tcc` を実行し、権限を許可し直してください。毎回これが必要になるのを避ける方法は「[署名用の証明書を作っておく](#署名用の証明書を作っておく)」を参照してください。
- パスワード入力などセキュア入力が有効な間は、キー入力を横取りできないため一時的にシステム標準の Cmd+Tab に戻ります。
- macOS 15 以降では、画面収録の許可について定期的に再承認を求めるバナーが表示されることがあります。
